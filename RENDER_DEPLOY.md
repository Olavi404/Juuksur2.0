# Render Deployment Guide (Laravel + PostgreSQL)

## Important First Check
Render can deploy this project only after the repository contains a full Laravel application root, including at least:
- `artisan`
- `composer.json`
- `package.json`
- `config/`, `public/`, `storage/`, and `vendor/` generated during build

If these are missing, first scaffold a fresh Laravel 12 app locally and copy the project files into it, then push again.

## 1. Prepare Local Project
Run locally in the repository root:

```bash
composer create-project laravel/laravel .
composer require laravel/breeze --dev
php artisan breeze:install blade
npm install
npm run build
```

Then keep the MVP files from this repo and commit everything.

## 2. Push to GitHub
```bash
git add -A
git commit -m "Prepare full Laravel app for Render"
git push
```

## 3. Deploy with Blueprint
1. Open Render dashboard.
2. Click `New +` -> `Blueprint`.
3. Connect repository `Olavi404/Juuksur2.0`.
4. Render reads `render.yaml` and creates:
   - one web service (`juuksur-app`)
   - one PostgreSQL database (`juuksur-db`)

## 4. Verify Environment
After first deploy, set or verify:
- `APP_URL` = your Render web service URL
- `APP_ENV=production`
- `APP_DEBUG=false`

`APP_KEY` is generated by blueprint.

## 5. First Startup Checks
After deploy completes:
1. Open service logs and verify no boot errors.
2. Visit `/` and confirm booking form loads.
3. Visit `/login` and log in with seeded admin account.
4. Visit `/admin/bookings` and verify protected access.

## 7. Troubleshooting
- `composer: command not found` or `artisan missing`:
  - repository does not yet contain full Laravel app root.
- Static asset issues:
  - ensure `npm ci` and `npm run build` succeeded in build logs.
- Database connection errors:
  - check DB env vars are linked from `juuksur-db`.
- Migration errors:
  - verify `preDeployCommand` ran successfully in deploy logs.
